# ROADMAP — «Баланс жизни» (Life Balance)

> Документ актуален на: **5 июля 2026**  
> Проект: `life-balance` — mobile-first веб-приложение для здоровья, питания и самочувствия.

---

## Обзор проекта

| Параметр | Значение |
|----------|----------|
| **Стек** | React 19, TypeScript, TanStack Start/Router, Vite, Tailwind 4, shadcn/ui |
| **Данные** | Supabase (Auth + PostgreSQL) |
| **AI** | OpenAI API (server-side, `gpt-4o-mini`) |
| **Графики** | Recharts |
| **Платформа** | Веб (PWA/APK/Capacitor — не подключены) |
| **Создание** | Lovable + Cursor (синхронизация с Lovable — см. `AGENTS.md`) |

## 1. Что уже реализовано

### 1.1. Аутентификация и онбординг ✅

- Регистрация, вход, выход, сброс пароля
- Создание профиля в Supabase при регистрации
- Согласия на юридические документы (`legal_consents`)
- Dev-утилиты отладки auth (`auth-debug.functions.ts`)

### 1.2. Голосовой дневник ✅ (приоритет 1 — выполнен)

- Web Speech API (русский язык, continuous mode, interim text)
- AI-извлечение данных из текста (`analyzeDayText` → structured JSON)
- Zod-валидация и безопасный парсинг (`ai-day-analysis.ts`)
- Сохранение в дневник с merge (не затирает ручной ввод)
- Страница `/voice` с подсказками и примерами фраз
- Модальное окно на главной (`VoiceDayDialog`)

### 1.3. Дневник питания ✅ (приоритет 2 — выполнен)

- Полная форма дня: приёмы пищи, напитки, вес, сон, шаги, настроение
- CRUD через Supabase (`daily_entries`)
- AI-анализ каждого приёма пищи (`analyzeMeal` + профиль)
- Быстрый ввод: шаблоны блюд (localStorage), копирование вчерашнего дня
- Надёжное сохранение: rollback, индикатор `saving`, toast при ошибках

### 1.4. Дневник здоровья ✅ (приоритет 3 — базово)

- Ежедневные показатели: давление, пульс, энергия, симптомы (`health_entries`)
- Особенности здоровья: хронические заболевания, ограничения, ЛФК (`health_features`)
- Привычки: курение, алкоголь, кофе, стресс, экранное время (`habits`)
- Merge daily + health записей на клиенте (`store.ts`)

### 1.5. AI-анализ ⚠️ (приоритет 4 — частично)

| Функция | Статус |
|---------|--------|
| `analyzeDayText` — анализ дня (голос/текст) | ✅ Работает |
| `analyzeMeal` — анализ приёма пищи | ✅ Работает |
| Страница `/ai` — персональные рекомендации | ❌ Заглушка «Скоро» |
| Ежедневный AI-отчёт на главной | ❌ Не реализован |
| Недельный AI-обзор | ❌ Не реализован |

### 1.6. Графики ✅ (приоритет 5 — базово)

- 8 графиков: вес, вода, сон, давление, пульс, энергия, хлебные единицы, кофе/сладкие напитки
- Данные из `useEntries()`
- Фильтр по периоду (7/30/90 дней) — **не реализован**

### 1.7. Профиль ✅ (приоритет 6 — с багами)

- Редактирование: имя, дата рождения, пол, рост, вес, цель
- Ссылки на: рецепты, особенности здоровья, привычки, настройки
- **Известные баги:** пустой экран без профиля в БД; «Удалить данные» чистит localStorage, а не Supabase

### 1.8. Рецепты ✅ (этап 1 — выполнен)

- Каталог из 10 статических рецептов
- Фильтры: заболевание, цель, время, ингредиенты
- Карточка и детальная страница (БЖУ, ингредients, шаги)
- Архитектура под AI-подбор, меню, список покупок (типы готовы)
- AI, фото, меню — **не подключены**

## 5. Подключённые API и сервисы

| Сервис | Назначение | Где используется | Переменные окружения |
|--------|------------|------------------|----------------------|
| **Supabase Auth** | Регистрация, вход, сессии | `src/lib/auth.ts`, все `_app` routes | `VITE_SUPABASE_URL`, `VITE_SUPABASE_PUBLISHABLE_KEY` |
| **Supabase PostgreSQL** | Профиль, дневник, здоровье, привычки | `src/lib/store.ts` | те же + RLS |
| **OpenAI Chat Completions** | AI-анализ дня и приёма пищи | `src/lib/ai.functions.ts` | `OPENAI_API_KEY`, `OPENAI_MODEL` |
| **Web Speech API** | Распознавание речи в браузере | `src/hooks/use-speech-recognition.ts` | HTTPS + микрофон (без API-ключа) |
| **Lovable Events** | Отчёт об ошибках | `src/lib/lovable-error-reporting.ts` | — |

### Таблицы Supabase (используются)

- `profiles` — профиль пользователя
- `daily_entries` — дневник питания/активности
- `health_entries` — давление, симптомы
- `habits` — привычки (1 запись на user)
- `health_features` — особенности здоровья (1 запись на user)
- `legal_consents` — согласия при регистрации (только запись)

### Не используются из Supabase

- Storage (файлы, фото)
- Edge Functions
- Realtime subscriptions

## 7. Риски архитектуры

### Критичные

| Риск | Описание | Митигация |
|------|----------|-----------|
| **Удаление данных в профиле** | Кнопка «Удалить все данные» чистит localStorage, не Supabase | Переписать на DELETE из Supabase |
| **Пустой профиль** | `if (!profile) return null` — белый экран | Показывать форму создания с `DEFAULT_PROFILE` |
| **Зависимость от OpenAI** | Без ключа AI недоступен; нет fallback | Кэш, offline-режим, graceful degradation (частично есть) |

### Средние

| Риск | Описание | Митигация |
|------|----------|-----------|
| **Merge daily + health на клиенте** | Две таблицы объединяются в `useEntries()` — сложность при partial save | Документировать контракт; тесты на merge |
| **Optimistic update** | UI обновляется до ответа Supabase (rollback есть, но не везде) | Единый паттерн через `saveEntry()` |
| **Web Speech API** | Только Chrome/Edge, HTTPS; iOS Safari ограничен | Whisper API как fallback |
| **Статические рецепты/recovery** | Не в БД — нет персонализации без миграции | `RecipeCatalogService` — контракт для замены |
| **i18n фрагментирован** | RU hardcoded + DOM-translator | Постепенный переход на словарь/i18next |
| **Orphan routes** | `/voice`, `/ai`, `/admin`, `/subscription` без BottomNav | Добавить в навигацию или профиль |

### Низкие / организационные

| Риск | Описание | Митигация |
|------|----------|-----------|
| **Lovable sync** | Force-push ломает историю на Lovable | Следовать `AGENTS.md` |
| **Мёртвый код** | `aiStubs.ts` не импортируется | Удалить при рефакторинге |
| **Юр. документы** | Демо-тексты в production | Юридическая экспертиза |
| **Нет тестов** | Регрессии при изменениях store/AI | Добавить unit-тests для парсеров и merge |
| **Размер бандла** | Recharts + router > 500 KB | Code splitting по маршрутам |

## 8. Какие функции лучше делать следующими

### Рекомендация (ближайшие 2–3 спринта)

1. **AI-подбор рецептов (этап 2 модуля «Рецепты»)**  
   Использовать готовые типы, `suggestRecipes()`, профиль и `analyzeMeal`-паттерн. Высокая ценность при минимальных архитектурных изменениях.

2. **Меню на день из рецептов**  
   Типы `DailyMenuPlan` уже есть. Связка с дневником питания — логичное продолжение.

3. **Страница AI-рекомендаций (`/ai`)**  
   Агрегация данных дня + профиль + `analyzeDayText` / новая server function `analyzeDaySummary`.

4. **Исправление багов профиля**  
   Быстрые wins: пустой экран, удаление из Supabase. Снижает риск для пользователей.

5. **Фильтры графиков (7/30/90 дней)**  
   Небольшой объём, заметное улучшение UX.

### Отложить до стабилизации MVP

- Android APK / Capacitor (после PWA)
- Подписка и платежи
- Админ-панель
- Apple Health / Google Fit
- Whisper API (если Web Speech достаточен для целевой аудитории)

### Связи между модулями (будущая ценность)

```
Голосовой дневник ──→ AI structured ──→ Дневник / Здоровье / Графики
       │
       └──→ AI-отчёт (/ai) ──→ Главная «Совет дня»

Рецепты ──→ Меню на день ──→ Дневник питания
       │
       └──→ Список покупок

Профиль (цель, заболевания) ──→ Фильтры рецептов + AI-рекомендации
```
